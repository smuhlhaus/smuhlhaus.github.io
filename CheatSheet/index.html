Positions Reference Table








The Rules

The purpose behind this table it to list, explain and demonstrate the different position attributes and properties an element can have. In the below table I have given all examples their own property while including a top: 20px rule to see how it affects element. Additionally, absolute was given a background colour of white, z-index of 1 and opacity of 0.9 to better acknowledge its location before scrolling.

Positions Purpose Properties Example Explanation
Static Default value. The element is positioned according to the normal flow of the document Properties do not work on this value; top, right, bottom, left, z-index Static Element stays in place
Relative In normal flow of document, but then offset from itself based on the values. Space is still created for the element. top, right, bottom, left, z-index Relative Element starts centered, but moves 20px away from top of box it is in. Can leave box.
Absolute Element is removed from normal flow of page and no space created for the element. Positioned relative to closest ancestor or otherwise containing block. top, right, bottom, left, z-index Absolute Element moves 20px down from top of page and stays in that specific location, can be hidden with scroll.
Fixed The element is removed from normal flow of page and nos space created in layout. Position is relative to initial containing block. top, right, bottom, left, z-index Fixed Like Absolute, Fixed comes down 20px from top of page, but is pinned on that point regardless of scroll. Used for nav bars.
Sticky Element is position in normal flow of page. Then offset to nearest scrolling ancestor and block. top, right, bottom, left, z-index Sticky The Sticky element will scroll with page until it meets the nearest scrolling block, which is the nav bar.
